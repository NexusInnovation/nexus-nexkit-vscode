## PR #158 Review — Danny

Danny reviewed PR #158 (Feature/invisible-nexkit).

**Verdict: APPROVED WITH NOTES**

PR moves `chat.*Locations` settings from workspace-level to user-level (`ConfigurationTarget.Global`) using `~/`-relative paths. `UserDirectoryService` is confirmed wired correctly. Overall implementation is clean, non-destructive, and cross-platform.

**3 minor issues noted:**
1. `_toTildePath()` silent fallback for paths outside `os.homedir()` should emit a warning log — failure mode is invisible otherwise.
2. `_isNexkitManagedPath()` does not detect backslash-format absolute paths (`C:\Users\...\\.nexkit\`) — low risk in practice (VS Code JSON uses forward slashes) but a known gap.
3. Writing workspace-relative `.nexkit/${subdir}` to `ConfigurationTarget.Global` for workspace overrides is unverified VS Code behaviour — relative paths in global scope may not resolve from workspace root. Should be validated or moved to `ConfigurationTarget.Workspace`.

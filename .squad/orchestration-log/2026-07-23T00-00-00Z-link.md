## Task

Fix accented-character (mojibake) corruption in the Convert to Markdown feature on Windows.

## Why Routed to Link

Link owns the `MarkitdownConversionService` and the broader `convert-to-markdown` feature (see decisions.md — Convert to Markdown implementation history), making them the correct owner for a bug fix in that service's subprocess handling.

## Root Cause

Python's `markitdown` subprocess defaults to the Windows locale codepage (e.g. cp1252) for stdout when output is piped rather than attached to a real console, instead of UTF-8. Node then mis-decoded the resulting single-byte accented characters as invalid UTF-8, producing one U+FFFD replacement character per accented letter.

## Fix

Added `PYTHONIOENCODING=utf-8` and `PYTHONUTF8=1` to the `spawn()` env for both `_runMarkitdown` and `_probeInterpreter` in `src/features/convert-to-markdown/markitdownConversionService.ts`.

## Task

Fix the failing test `ConvertToMarkdownPanelService > save-to-file`, caused by Sinon being unable to stub the non-configurable `vscode.workspace.fs.writeFile` property.

## Why Routed to Link

Link owns the `ConvertToMarkdownPanelService` / `MarkitdownConversionService` feature and its test suite (see decisions.md — Convert to Markdown implementation history), making them the correct owner for this test-infrastructure fix.

## Root Cause

`FileSystem` instance members (e.g. `vscode.workspace.fs.writeFile`) are frozen/non-configurable in the current VS Code test host, so `sinon.stub(vscode.workspace.fs, "writeFile")` throws a "non-configurable and non-writable" descriptor error. The `fs` getter on `vscode.workspace` itself, however, is configurable.

## Fix

Stubbed the `fs` getter on `vscode.workspace` with a spread copy overriding only `writeFile`, instead of stubbing the frozen method directly:

```ts
sinon.stub(vscode.workspace, "fs").value({ ...vscode.workspace.fs, writeFile: sinon.stub().resolves() });
```

## Root Cause & Analysis

RecommendedSettingsConfigDeployer.deployVscodeSettings() was being invoked automatically via startupVerification.verifyOnStartup() during extension activation. This method writes ConfigurationTarget.Global settings, which causes VS Code to automatically open the user profile settings.json file as a side effect.

## Changes

### Implementation

- Moved user-level settings writes out of automatic activation path
- User settings now only deployed during explicit workspace initialization (verifyOnInitialization)
- Added no-op write guard in RecommendedSettingsConfigDeployer to prevent unintended writes
- Preserved all automated verification for workspace-level settings

### Root Cause Fix

**Before:** Activation → startupVerification.verifyOnStartup() → RecommendedSettingsConfigDeployer.deployVscodeSettings() → ConfigurationTarget.Global write → VS Code opens settings.json  
**After:** Activation → startupVerification.verifyOnStartup() (workspace checks only) → [explicit init only] → RecommendedSettingsConfigDeployer.deployVscodeSettings()

## Outcomes

All 358 tests passing. User profile settings no longer opened spuriously on extension startup while maintaining verification integrity.

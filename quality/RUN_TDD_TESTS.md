# TDD Verification Protocol: nexus-nexkit-vscode

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-07-24 · Project: nexus-nexkit-vscode

## Purpose

For each confirmed bug, prove reproducibility and fix correctness with red/green execution evidence.

## Per-Bug Flow

1. Read bug entry in quality/BUGS.md and mapped requirement in quality/REQUIREMENTS.md.
2. Ensure regression patch exists at quality/patches/BUG-NNN-regression-test.patch.
3. Red phase:

- Temporarily disable skip/xfail guard.
- Run bug-specific regression test on unpatched code.
- Expect fail.
- Write quality/results/BUG-NNN.red.log with status tag RED.

4. Green phase (if fix patch exists):

- Apply quality/patches/BUG-NNN-fix.patch.
- Run same test.
- Expect pass.
- Write quality/results/BUG-NNN.green.log with status tag GREEN.

5. Restore guard if fix is not permanently merged in source.

## Language-Aware Commands

- TypeScript/JavaScript (Mocha): npm run test:unit -- --grep "BUG-NNN"
- TypeScript/JavaScript (Jest): npx jest --testNamePattern "BUG-NNN"
- TypeScript/JavaScript (Vitest): npx vitest run --testNamePattern "BUG-NNN"

## Traceability Output

Write quality/TDD_TRACEABILITY.md with:
| Bug ID | Requirement ID | Spec Doc | Spec Section | Behavioral Contract | Test File:Function | Red Result | Green Result |

Required per bug entry fields:

- id
- requirement
- red_phase
- green_phase
- verdict
- fix_patch_present
- writeup_path

## Verdict Rules

- TDD verified: red fail then green pass.
- confirmed open: red fail and no fix patch.
- red failed: regression test did not fail on unpatched code.
- green failed: regression still fails after fix patch.
- deferred: test could not run due environment limits (document in notes).

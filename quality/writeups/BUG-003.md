# BUG-003 Writeup

## Summary

Managed-file rollback/restore uses UTF-8 string serialization rather than byte-preserving I/O.

## Requirement Trace

- REQ-004 (Byte-Preserving Change Rollback)

## Evidence

- src/features/nexkit-file-watcher/nexkitFileWatcherService.ts:195
- src/features/nexkit-file-watcher/nexkitFileWatcherService.ts:207
- src/features/nexkit-file-watcher/nexkitFileWatcherService.ts:291

## Reproduction

1. Place non-UTF8 content in a managed file under .nexkit.
2. Trigger external change/delete restore flow.
3. Observe content round-trip via UTF-8 text path, risking byte drift.

## TDD Status

- Red phase: FAIL reproduced (expected) in quality/results/BUG-003.red.log.
- Green phase: Not executed (no fix patch present).
- Verdict: confirmed open.

## Proposed Fix Shape

Cache and restore Buffer values for managed files and use byte-level equality checks for rollback.

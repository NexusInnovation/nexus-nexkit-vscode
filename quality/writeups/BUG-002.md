# BUG-002 Writeup

## Summary

MCP config parse/read failure falls back to a fresh object and then writes output, enabling destructive overwrite.

## Requirement Trace

- REQ-006 (Parse Failure Preservation)
- REQ-007 (Cross-Surface Merge Consistency)

## Evidence

- src/features/mcp-management/mcpConfigService.ts:142
- src/features/mcp-management/mcpConfigService.ts:144
- src/features/mcp-management/mcpConfigService.ts:189

## Reproduction

1. Introduce malformed JSON in user or workspace mcp.json.
2. Trigger add/update MCP server path.
3. Observe config rewritten from fallback baseline instead of preserving original bytes.

## TDD Status

- Red phase: FAIL reproduced (expected) in quality/results/BUG-002.red.log.
- Green phase: Not executed (no fix patch present).
- Verdict: confirmed open.

## Proposed Fix Shape

On parse/read error, abort write path and return actionable error while preserving existing file content.

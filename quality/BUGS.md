## BUG-001: Startup auth prompt is triggered during passive startup

- Severity: High
- Requirement: REQ-003, REQ-005
- Evidence:
  - src/features/initialization/startupVerificationService.ts:41 unconditionally calls ensureAuthenticated() in verifyOnStartup().
  - src/features/initialization/githubAuthPromptService.ts:36 shows warning prompt when no session/token is found.
- Risk:
  - Opening a workspace can trigger an auth interruption even when user did not invoke a GitHub-dependent action.
- Expected behavior:
  - Startup should remain non-interactive; auth prompt only on explicit user-triggered operations requiring GitHub.
- Closure status: Open
- Regression test: quality/test_regression.js :: BUG-001

## BUG-002: MCP config parse failures can cause destructive overwrite

- Severity: Critical
- Requirement: REQ-006, REQ-007
- Evidence:
  - src/features/mcp-management/mcpConfigService.ts:142 parses existing config in try block.
  - src/features/mcp-management/mcpConfigService.ts:144 catches all parse/read errors and continues with default config.
  - src/features/mcp-management/mcpConfigService.ts:189 writes JSON back regardless of parse failure path.
- Risk:
  - A malformed config file can be replaced with a fresh object, losing unrelated existing entries.
- Expected behavior:
  - Parse/read errors should fail safely and preserve original file bytes.
- Closure status: Open
- Regression test: quality/test_regression.js :: BUG-002

## BUG-003: Managed file restore path is UTF-8 only and not byte-safe

- Severity: High
- Requirement: REQ-004
- Evidence:
  - src/features/nexkit-file-watcher/nexkitFileWatcherService.ts:195 reads user content with utf8.
  - src/features/nexkit-file-watcher/nexkitFileWatcherService.ts:207 writes original content with utf8.
  - src/features/nexkit-file-watcher/nexkitFileWatcherService.ts:291 writes restored deleted content with utf8.
- Risk:
  - Non-UTF8 managed files can be corrupted during rollback/restore flow.
- Expected behavior:
  - Preserve byte-exact content using Buffer-based reads/writes where data fidelity matters.
- Closure status: Open
- Regression test: quality/test_regression.js :: BUG-003

## BUG-004: Workflow script command arguments are interpolated without escaping

- Severity: High
- Requirement: REQ-008
- Evidence:
  - src/features/github-workflow-runner/githubWorkflowRunnerService.ts:417 constructs PowerShell args via string interpolation.
  - src/features/github-workflow-runner/githubWorkflowRunnerService.ts:436 constructs shell args via string interpolation.
  - src/features/github-workflow-runner/githubWorkflowRunnerService.ts:217 sends raw command string to terminal.
- Risk:
  - Crafted event/job values containing quotes/control chars can alter argument boundaries and execute unintended semantics.
- Expected behavior:
  - Use shell-safe escaping or argument-array APIs that preserve boundaries for both PowerShell and POSIX.
- Closure status: Open
- Regression test: quality/test_regression.js :: BUG-004

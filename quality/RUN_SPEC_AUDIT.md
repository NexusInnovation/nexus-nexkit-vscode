# Spec Audit Protocol: nexus-nexkit-vscode

> Generated by [Quality Playbook](https://github.com/andrewstellman/quality-playbook) v1.5.3 — Andrew Stellman
> Date: 2026-07-24 · Project: nexus-nexkit-vscode

## Council Setup

Run three independent audits against quality/REQUIREMENTS.md and source code.
Each auditor performs a full audit; optional secondary emphasis lenses:

- Auditor 1: lifecycle and startup side effects
- Auditor 2: configuration/merge/data-preservation behaviors
- Auditor 3: execution boundary and command construction safety

## Pre-Audit Docs Validation

reference_docs/ is absent for this run. Baseline evidence is Tier-3 source code plus repository docs/README.

## Auditor Prompt Skeleton

For each requirement REQ-001..REQ-012:

- Verdict: SATISFIED / VIOLATED / PARTIALLY SATISFIED / UNCLEAR
- Evidence: file:line citations
- Reasoning: expected vs actual behavior
- Candidate bug if violated

Do not accept requirement satisfaction without code evidence.

## Triage Protocol

1. Merge findings by requirement ID and code location.
2. Any minority finding (1/3 auditors) requires explicit re-investigation.
3. Resolve conflicts with executable verification probes.

## Verification Probes

For each disputed finding, write executable assertion in quality/spec_audits/triage_probes.sh:

- Confirmations must fail on current code.
- Rejections must pass with line-anchored evidence from source.

## Output Artifacts

- quality/spec_audits/YYYY-MM-DD-auditor-1.md
- quality/spec_audits/YYYY-MM-DD-auditor-2.md
- quality/spec_audits/YYYY-MM-DD-auditor-3.md
- quality/spec_audits/YYYY-MM-DD-triage.md
- quality/spec_audits/triage_probes.sh

## Layer 2 Citation Semantic Check

If Tier 1/2 requirements exist, produce quality/citation_semantic_check.json with per-REQ reviewer verdicts.
If none exist, still write the file with empty reviews array.

## Net-New Bug Handling

Every confirmed code bug from triage must be added to quality/BUGS.md and mapped to a regression test in quality/test_regression.\*.
